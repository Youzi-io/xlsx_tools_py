# XLSX 复杂表头分析工具 — 实现原理说明

> 版本: v1.0.0 | 代码总量: ~4000 行 | Python + PyQt5 桌面应用

---

## 一、项目概述

本工具是一个**本地桌面程序**，用于分析带有**复杂表头**（多行表头、合并单元格）的 `.xlsx` 文件。用户可通过 **SQL 查询**或**可视化条件构建器**对数据进行筛选、分组、统计，结果以可交互表格展示并支持多格式导出。

### 核心能力

| 能力 | 说明 |
|------|------|
| 复杂表头识别 | 自动检测多行/多层级/合并单元格表头，生成扁平化列名 |
| SQL 查询 | 对 Excel 数据执行标准 SQL（SELECT/WHERE/GROUP BY/JOIN/窗口函数） |
| 条件构建器 | 无需 SQL 知识，通过下拉选择和输入框构建查询 |
| 数据统计 | 分组聚合、透视表、频率分析、Top-N、相关性矩阵 |
| 结果导出 | xlsx / csv |

## 三、核心模块原理

### 3.1 复杂表头识别引擎 (`header_detector.py`)

这是整个工具最核心的模块，解决"Excel 表头不是简单的一行"这一问题。

#### 面临的挑战

```
┌──────────────────────────────────────────────────────┐
│                  2024年度财务报表                ← 合并单元格跨4列  │
├──────────┬──────────┬──────────┬────────────────────┤
│ 基本信息  │ 基本信息  │    Q1    │       Q2           │  ← 第2行也有合并
├────┬────┼────┬────┼────┬────┼────┬──────────────┤
│姓名│部门│收入│支出│收入│支出│                      │  ← 第3行细节
├────┼────┼────┼────┼────┼────┼──────────────────┤
│张三│技术 │ 50K│ 30K│ 60K│ 35K│ ...               │  ← 数据行
└────┴────┴────┴────┴────┴────┴──────────────────┘
```

用户期望的列名：`2024年度财务报表_Q1_收入`、`基本信息_姓名` 等。

#### 识别算法：多维度加权评分

对前 50 行逐行打分，判断是否为表头行：

```
行得分 = 格式得分×35% + 类型得分×35% + 关键词得分×20% + 合并得分×10%
```

| 维度 | 权重 | 判断依据 |
|------|------|---------|
| 格式得分 | 35% | 单元格是否加粗 (`font.bold`)、是否有背景色 (`fill.patternType`) |
| 类型得分 | 35% | 该行中字符串值占比，越高越像表头 |
| 关键词得分 | 20% | 是否包含"序号""姓名""部门""合计"等表头关键词 |
| 合并得分 | 10% | 该行有多少单元格属于合并区域 |

**边界判定**：用滑动窗口找连续高分区域，窗口内得分方差小（稳定）额外加分。

**置信度**：表头行平均分 - 首个数据行分，差异越大置信度越高。

#### 合并单元格展开

```
合并区域 A1:B1，值 "基本信息"
  → 展开为 A1="基本信息", B1="基本信息"（值填充到所有被合并的单元格）
```

实现函数 `expand_merged_value()`，遍历 `ws.merged_cells.ranges` 判断每个坐标是否属于合并区域。

#### 层级列名构建

```
第1行: 基本信息  /  2024年度  /  2024年度   ← 合并单元格展开后
第2行: 姓名      /  部门      /  Q1        /  Q2
第3行: (空)     /  (空)      /  收入       /  支出

→ 列名: 基本信息_姓名, 基本信息_部门, 2024年度_Q1_收入, 2024年度_Q2_支出
→ 层级: ["基本信息","姓名"], ["基本信息","部门"], ["2024年度","Q1","收入"]...
```

去重规则：连续重复值（合并单元格导致）去重；空值和 "Unnamed" 跳过。

### 3.2 SQL 查询引擎 (`query_engine.py`)

基于 **duckdb**，直接在 pandas DataFrame 上执行标准 SQL。

**为什么选 duckdb？**
- 零配置，无需数据库服务（SQLite 不行，TypeORM 太重）
- 原生支持对 pandas DataFrame 执行完整 SQL（JOIN、窗口函数、子查询）
- 列名支持双引号包裹中文

#### 工作流程

```
用户输入 SQL → 预处理(修复常见错误) → duckdb.execute() → pandas DataFrame
```

#### SQL 预处理：智能纠错

用户常犯的错误：用双引号写字符串值。

```sql
-- 错误写法（"广州分公司" 被当成列名）
WHERE 开票商名称 = "广州分公司"

-- 预处理自动修正
WHERE 开票商名称 = '广州分公司'
```

实现逻辑：正则匹配所有 `"..."` 内容，判断内容是否是 DataFrame 中存在的列名——是列名则保留双引号，否则替换为单引号。

#### 表结构生成

`get_schema_info()` 将 DataFrame 的列和类型映射为 `CREATE TABLE` 语句，展示给用户作为 SQL 参考：

```sql
CREATE TABLE data (
    "姓名" VARCHAR,
    "薪资" DOUBLE,
    "入职日期" TIMESTAMP,
);
```

### 3.3 可视化条件构建器 (`query_builder.py`)

将用户的可视化选择**翻译为 SQL**：

```
用户操作:  [列: 薪资 ▼] [运算符: 大于 ▼] [值: 10000]
         [+ 添加条件] [连接词: AND ▼]
         [列: 部门 ▼] [运算符: 等于 ▼] [值: 技术部]
         分组: [部门 ▼]  统计: [求和 ▼] [薪资 ▼]
         排序: [薪资 ▼] [降序 ▼]  限制: 100行

→ 生成的 SQL:
  SELECT "部门", SUM("薪资") FROM data
  WHERE "薪资" > 10000 AND "部门" = '技术部'
  GROUP BY "部门"
  ORDER BY "薪资" DESC LIMIT 100
```

**条件行 (`ConditionRow`)**：列选择 → 运算符 → 值输入 → 删除按钮。第一行不显示连接词，后续行显示 AND/OR。

**运算符**：等于、大于、小于、包含(LIKE)、为空、介于(BETWEEN) 等 11 种。值为纯数字时自动去掉引号。

### 3.4 XLSX 读取器 (`xlsx_reader.py`)

封装 openpyxl，`data_only=True` 模式读取（公式单元格读取计算结果而非公式）。提供：

- 安全打开/关闭（上下文管理器支持）
- Sheet 列表和信息获取
- 按行列读取数据
- 单元格样式提取（字体、背景色、边框），用于表头识别

---

### 3.5 统计引擎 (`stats_engine.py`)

封装 pandas 常用统计操作：

| 方法 | 功能 | pandas 实现 |
|------|------|-----------|
| `describe()` | 描述性统计 | `df.describe()` |
| `group_by()` | 分组聚合 | `df.groupby().agg()` |
| `pivot()` | 透视表 | `pd.pivot_table()` |
| `value_counts()` | 频率统计 | `df[col].value_counts()` |
| `correlation()` | 相关矩阵 | `df.corr()` |
| `top_n()` | Top-N 排名 | `df.nlargest()` / `df.nsmallest()` |
| `get_summary_stats()` | 单列详细统计 | 自定义计算 |

---

### 3.6 Pandas → Qt 表格适配 (`result_table.py`)

实现 `QAbstractTableModel` 子类，将 pandas DataFrame 映射为 Qt 表格：

```
PandasTableModel(QAbstractTableModel)
├── data()          → 单元格显示值（NaN→空、浮点数格式化）
├── headerData()    → 列名 + 行号
├── sort()          → 点击表头排序（pandas sort_values）
├── rowCount()      → len(df)
└── columnCount()   → len(df.columns)
```

## 四、数据流全景

```
用户打开 .xlsx 文件
    │
    ▼
XlsxReader 打开工作簿 (openpyxl, data_only=True)
    │
    ▼
HeaderDetector 扫描前 N 行
    ├─ 多维度评分 → 确定表头行范围
    ├─ 合并单元格展开 → 值填充
    ├─ 层级路径构建 → 扁平化列名
    └─ 输出 HeaderInfo(header_rows, data_start_row, columns)
    │
    ▼
DataManager 加载数据行 → pandas DataFrame
    ├─ 列名设为扁平化名称
    └─ 自动类型转换 (string → numeric)
    │
    ├──→ HeaderView 树形展示表头层级
    ├──→ FilePanel 展示列列表和数据摘要
    ├──→ SqlEditor 展示表结构 (CREATE TABLE)
    └──→ QueryBuilder 填充可用列下拉
    │
    ▼
用户执行查询
    │
    ├─ SQL 查询路径:
    │   SqlEditor → 信号 → MainWindow._execute_sql()
    │
    └─ 条件构建器路径:
        QueryBuilder → 生成 SQL → 信号 → MainWindow._execute_sql()
    │
    ▼
QueryEngine 预处理 SQL (双引号→单引号纠正)
    │
    ▼
duckdb 执行 SQL → 返回结果 DataFrame
    │
    ▼
ResultTable (PandasTableModel) 渲染表格
    │
    ▼
ExportDialog: 导出为 xlsx / csv
```

## 五、关键设计决策

### 1. duckdb 而非 SQLite
- SQLite 需要用 `CREATE TABLE` 先把数据写进去（列名含中文特殊字符很麻烦）
- duckdb 直接 `con.register("data", df)` 注册 DataFrame，零拷贝

### 2. PyQt5 而非 PyQt6
- 经用户要求，兼容 Windows 7（PyQt6 不在此系统上）
- `QAction` 在 PyQt5 的 `QtWidgets` 下；`Qt.Horizontal` 无需 `Orientation` 前缀

### 3. 表头识别用评分而非规则
- 硬编码"第1行就是表头"无法应对多行/合并表头
- 多维度评分自适应各种表头布局

### 4. core/gui 分层
- 核心引擎可脱离 GUI 使用
- 测试可直接测 core 模块（17 个单元测试）
